# SRE Portfolio - Implantação de Gestão de Incidentes e SLA

Este repositório contém o código-fonte de uma **Landing Page de Portfólio** desenvolvida para demonstrar experiência e resultados na área de SRE (Site Reliability Engineering) e Observabilidade. 

A página detalha um case de sucesso sobre a estruturação de processos de monitoramento corporativo, focado na redução de *downtime*, melhoria do MTTR (Mean Time to Repair) e excelência operacional.

## 🚀 Tecnologias Utilizadas

O projeto foi construído utilizando tecnologias modernas de front-end com foco em performance e responsividade:

- **HTML5**: Estruturação semântica do conteúdo.
- **Tailwind CSS**: Framework CSS utilitário para estilização rápida e responsiva (via CDN).
- **Google Fonts**: Utilização da fonte 'Inter' para tipografia moderna e legível.
- **Google Material Symbols**: Ícones escaláveis para enriquecer a interface visual.

## 📊 Estrutura do Case de Estudo

A landing page é dividida em seções que contam a história do projeto técnico:

1. **Contexto Operacional**: Apresenta o cenário inicial, ferramentas utilizadas (Jira Service, Zabbix, Grafana, Dynatrace) e os desafios enfrentados.
2. **Matriz de Prioridade e Resposta**: Demonstração de SLAs definidos (P1 a P4) baseados no impacto ao negócio.
3. **Dashboard de Observabilidade**: Visualização interativa de métricas essenciais como Conformidade de SLA Global, MTTR e Volume de Incidentes.
4. **Sucesso e Impacto no Negócio**: Resultados tangíveis como redução de falsos positivos e aumento na satisfação do usuário.
5. **Melhoria Contínua**: O ciclo de vida do projeto pós-implementação e integração com CI/CD.

## ⚙️ Como Executar o Projeto Localmente

Como o projeto é composto por arquivos estáticos e utiliza o Tailwind via CDN, não é necessário instalar dependências complexas.

1. Clone o repositório (ou faça o download dos arquivos):
   ```bash
   git clone https://github.com/SEU_USUARIO/SEU_REPOSITORIO.git
   ```
2. Navegue até a pasta do projeto:
   ```bash
   cd portifolio
   ```
3. Abra o arquivo `CRM.html` no seu navegador de preferência:
   - Dê um duplo clique no arquivo `CRM.html` 
   - OU arraste-o para dentro de uma aba do Google Chrome, Firefox, Edge, etc.

## 🌐 Publicação (Deploy)

Para colocar este portfólio no ar de forma gratuita, recomendamos o uso do **GitHub Pages**:

1. Envie seu código para o GitHub.
2. Acesse as **Configurações (Settings)** do seu repositório no GitHub.
3. No menu lateral esquerdo, clique em **Pages**.
4. Em *Build and deployment*, selecione a branch `main` como *Source*.
5. Salve e aguarde alguns minutos para acessar seu portfólio online.
